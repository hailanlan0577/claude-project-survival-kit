# 2. 七件套骨架详解

> 每个文件应该装什么、不装什么。

---

## 📋 总览表

| 文件 | 角色 | 长度（建议） | 变化频率 | 谁读 |
|------|------|-------------|---------|------|
| `ONBOARDING.md` | 🪄 开场白 | 100-150 行 | 🟡 偶尔 | 新 Claude |
| `OFFBOARDING.md` | 🛑 收场 checklist | 150-200 行 | 🟢 几乎不变 | 当前 Claude |
| `CLAUDE.md` | 🗺️ 地图 + 部署 | 200-300 行 | 🟡 偶尔 | Claude Code 自动加载 |
| `STATUS.md` | 📊 进度快照 | 滚动增长 | 🔴 最勤 | 所有人 |
| `RUNBOOK.md` | 🔧 场景救命 | 300-500 行 | 🟢 很少 | 用户 |
| `scripts/deploy.sh` | 🚀 部署脚本 | 80-120 行 | 🟢 很少 | 机器 |
| `.gitignore` | 🚫 入库黑名单 | 20-40 行 | 🟢 几乎不变 | git |

---

## 1. ONBOARDING.md — 开场白

### 这里装什么

✅ 项目地形（源码路径 / GitHub / 部署目标）
✅ N 大禁忌（踩过的坑）
✅ 项目是什么（一句话）
✅ Phase 完成度表
✅ 当前阻塞（根因 + 现象）
✅ A/B/C 修复路径（**不要让 Claude 自己选**）
✅ 文档读取优先顺序（防新 Claude 一上来读全量设计文档炸 context）
✅ 上次会话踩的坑（警示新 Claude）
✅ Claude 该做什么（汇报 + 等待）

### 这里不装

❌ 大段架构图（放设计文档）
❌ 完整代码地图（放 CLAUDE.md）
❌ 每天的进度（放 STATUS.md）
❌ 故障排查（放 RUNBOOK.md）

### 黄金标准

新 Claude 读完 60 秒内能说出：
1. 这个项目是做啥的
2. 卡在哪
3. 有哪几个选项可以推进
4. 该问用户什么问题

---

## 2. OFFBOARDING.md — 收场 checklist

### 这里装什么

✅ 触发时机（用户说什么自动触发）
✅ 8 步 checklist（STATUS / 禁忌 / commit / 部署 / 密钥 / 代码地图 / 记忆 / 便条）
✅ Claude 自检信号（什么时候主动提议收尾）
✅ 完整会话生命周期图

### 这里不装

❌ 具体项目信息（这是"方法论"文档，跨项目可复用）
❌ 具体命令（命令放 RUNBOOK，这里只给"要做什么"）

### 黄金标准

Claude 读完能**一步步跑完**，每步都知道具体要改哪个文件、跑什么命令。

---

## 3. CLAUDE.md — Claude Code 自动加载

### 这里装什么

Claude Code 启动时会**自动读**这个文件。所以这里放：

✅ 指向 ONBOARDING.md 的指引（顶部）
✅ 地形（源码位置 vs 部署目标，禁忌）
✅ 部署工作流（分步命令）
✅ 远程访问姿势（SSH 别名 + 认证方式）
✅ 关键服务端点（端口 / 容器名 / 数据卷）
✅ 技术栈清单
✅ 代码地图（每个目录/文件干啥）
✅ 健康检查命令
✅ 敏感文件清单
✅ 权威文档位置表

### 这里不装

❌ 设计决策背景（放 Obsidian / 设计文档）
❌ 每日进度（放 STATUS.md）

### 黄金标准

Claude Code 读完 `CLAUDE.md` 知道：
1. 代码在哪
2. 改完怎么部署
3. 有哪些服务要尊重（别碰别人的 Qdrant）
4. 遇到什么查 RUNBOOK 或 STATUS

---

## 4. STATUS.md — 进度快照

### 这里装什么

✅ 一句话总结当前阻塞
✅ Phase 完成度表（每 Phase 子任务状态）
✅ 当前阻塞根因分析
✅ 备选修复路径 A/B/C
✅ 其他已知小问题
✅ "YYYY-MM-DD 做了什么"按日期追加（保留历史）
✅ 下一步建议顺序
✅ **末尾**："🎯 下次进来第一件事"给下个 Claude 留便条

### 这里不装

❌ 详细命令（放 RUNBOOK）
❌ 架构图（放设计文档）

### 黄金标准

- 用户打开能看到"现在走到哪"
- 新 Claude 打开知道从哪继续
- 每次会话结束追加**结果式**记录（不写过程）

### 追加格式

```markdown
## 📝 YYYY-MM-DD 做了什么

1. ✅ **修了 X** — median 现在是 Y
2. ⚠️ **飞书卡片没做完** — API 文档看了一半，下次接着 Phase 3
3. ❌ **尝试加品牌 LLM 标准化** — qwen-turbo 响应 3-5s 太慢，暂时还用字符串匹配
```

## 5. RUNBOOK.md — 场景救命

### 这里装什么

**场景驱动的 9 章**（+ 1 章 git 速查）：

1. 会话交接（指向 ONBOARDING）
2. 日常部署（`deploy.sh` 用法 + 回滚）
3. 服务故障排查（进程/容器/日志）
4. 从零重建（换机器 / 重装系统）
5. 备份恢复（数据库 / 资产 / config）
6. 密钥恢复（指向 credentials.md）
7. 健康检查一键脚本
8. 定期维护频率表
9. 外部依赖清单
10. Git 日常速查（10 个场景，含 🚨 密钥推错紧急处理）

### 这里不装

❌ 项目设计背景
❌ 每日进度

### 黄金标准

**场景索引表**永远在文件顶部。用户出问题第一时间查这个表，找到章节跳过去。

---

## 6. scripts/deploy.sh — 一键部署

### 这里装什么

✅ 三种模式：full / --dry-run / --restart
✅ MD5 完整性校验（防 scp 截断）
✅ 上一版自动备份（`server.prev` 可回滚）
✅ 彩色输出（用户能看懂进度）
✅ 失败立即退出（`set -e`）
✅ 回滚命令提示（脚本末尾打印）

### 这里不装

❌ 业务逻辑（这是部署脚本，不是应用代码）
❌ 敏感信息（SSH key、密码都走系统 keychain）

---

## 7. .gitignore — 入库黑名单

### 必须忽略

- `configs/config.yaml` / `.env` / `.env.local`（真实密钥）
- `bin/` / `dist/` / `build/`（编译产物）
- `node_modules/` / `vendor/` / `.venv/`（依赖）
- `data/` / `*.log` / `*.sqlite`（运行时数据）
- `.DS_Store` / `.vscode/` / `.idea/`（系统/IDE）

### 按需忽略

- `deploy/*.bin`（救命 binary 副本太大不入库，异地备份）
- `.omc/state/`（OMC 运行时状态）

### 黄金标准

每次 `git init` 前 **先写 `.gitignore`**，然后 `git status --ignored --short` 确认敏感文件在 `!!` 列表。
