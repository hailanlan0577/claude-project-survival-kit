# 4. 备份分层策略

> **不是所有东西都值得备份**。过度备份浪费磁盘，漏掉关键资产掉链子。
>
> 这一篇帮你分清楚：**哪些要 / 哪些不用 / 每多久一次**。

---

## 📊 完整分层

| 资产类型 | 策略 | 频率 | 为什么 |
|---------|------|------|--------|
| 📝 **源码** | GitHub + 本地 + 部署目标 | 每次改动 | 最宝贵的资产 |
| ⚙️ **配置** | credentials.md 全量 + 两地 config 校验 | 每次改密钥 | 丢了服务跑不起来 |
| 🗃️ **数据库** | 定期 snapshot | 每周 | 业务核心 |
| 🖼️ **永久图片/资产** | 打包到 backup 目录 | 每次入库后 | 原源可能消失 |
| 🗑️ **临时图片/缓存** | **不备份** | — | 能从源重建 |
| ⚙️ **运行 binary** | 本地 deploy/ + 异地（iCloud/坚果云） | 每次稳定里程碑 | 源码改了也有活版本顶上 |
| 📋 **服务定义** | LaunchAgent plist / systemd unit 入 git | 每次修改 | 系统崩了能复活 |

---

## 🔍 每类资产详解

### 1. 源码（三份活的副本）

**策略**：**三地同步**

```
本地 MacBook ─────git push──────→ GitHub
     │                                │
     │ rsync                          │ clone/pull
     ▼                                ▼
部署目标 / 服务器（部署镜像，可滞后）
```

**备份频率**：每次改动（commit + push）。

**为什么三份**：
- GitHub 挂了 → 本地还有
- 本地硬盘坏了 → GitHub 能 clone 回来
- 误删文件 → git 历史能找回

**自检**：
```bash
git status --short         # 有没有未 commit
git log origin/main..HEAD  # 有没有未 push
```

---

### 2. 配置（密钥敏感）

**策略**：**三位一体**（见 `docs/3-密钥管理三位一体.md`）

1. `configs/config.yaml` — 本地真实（不入库）
2. `configs/config.example.yaml` — 入库脱敏
3. `~/.claude/.../credentials.md` — 全量备份

**备份频率**：每次改密钥或加新服务。

**自检**：
```bash
# 两地一致性
diff <(md5 -q <本地 config>) <(ssh <服务器> 'md5 -q <远程 config>')
```

---

### 3. 数据库（snapshot）

**策略**：**定期自动备份 + 保留最近 N 版**

**Qdrant 示例**：
```bash
NAME=$(curl -s -X POST http://127.0.0.1:6335/collections/<col>/snapshots | python3 -c "import sys,json;print(json.load(sys.stdin)['result']['name'])")
curl -s -o ~/backups/$NAME http://127.0.0.1:6335/collections/<col>/snapshots/$NAME
```

**Postgres 示例**：
```bash
pg_dump -U <user> <dbname> | gzip > ~/backups/db-$(date +%Y%m%d).sql.gz
```

**备份频率**：每周（Phase 1 入库新数据后立即一次）。

**保留策略**：
- 最近 4 个快照
- 每月留一个月度快照
- 每年留一个年度快照

---

### 4. 永久资产 vs 临时资产

这是**最容易判断错**的地方。

#### ✅ 永久资产（必须备份）

- **从第三方 CDN 下载并本地化的图片**（CDN 过期会 404）
- **用户上传的文件**
- **AI 生成结果**（调一次 API 花钱了）

#### ❌ 临时资产（不备份）

- **爬虫/API 抓来的缓存图片**（URL 还在，能重抓）
- **中间计算结果**（能重算）
- **日志文件**（滚动覆盖即可）

#### 实战例子：`luxury-bag-copilot` 11GB 图片

```
~/luxury-bags-images/
├── 经验库图（从奢当家 Excel 下载）      ← 永久资产，165 张，备份
└── 爬虫缓存图（每次 poller 下载）       ← 临时资产，18000+ 张，不备份
```

**只备份经验库那 165 张（33MB），不备份其他 11GB 爬虫缓存**。因为：
- 爬虫图 URL 还在闲鱼数据库
- 向量早就进了 Qdrant（图本身不再影响检索）
- 备份 11GB 没意义，占存储

**教训**：区分资产类型，别懒得分就全备份。

---

### 5. 运行 binary（黄金副本）

**策略**：**本地 + 异地两处**

部署目标上跑着的 binary 是**当前在线业务的根本**。如果源码有未提交改动、或源码丢了，这个 binary 是唯一"活版本"。

**本地副本**：
```bash
scp -C <部署目标>:<binary 路径> <本地>/deploy/server-<日期>.bin
```

**异地副本**（推荐）：
- iCloud Drive / 坚果云：拖到 `~/Library/Mobile Documents/com~apple~CloudDocs/backups/`
- 网盘：百度云 / Google Drive

**备份频率**：每次稳定里程碑（部署+跑稳后）。

**不入 git**：binary 20-50MB，git 不适合存大二进制。在 `.gitignore` 里加 `deploy/*.bin`。

---

### 6. 服务定义文件

**策略**：**入 git，放 `deploy/` 目录**

- macOS LaunchAgent：`deploy/<label>.plist`
- Linux systemd：`deploy/<service>.service`
- Docker Compose：`deploy/docker-compose.yml`

这些是 kilobytes 级别的配置文件，入 git 没负担。

**备份频率**：每次修改服务定义（改环境变量、日志路径、资源限制等）。

**从零重建时**：克隆仓库 → `cp deploy/*.plist ~/Library/LaunchAgents/` → `launchctl load`。

## 🗓️ 推荐的维护节奏

### 每次会话结束（由 `OFFBOARDING.md` 保证）

- ✅ git commit + push
- ✅ STATUS.md 追加
- ✅ 记忆系统存进度

### 每次部署成功

- ✅ binary 自动保留上一版（`server.prev`）
- ✅ MD5 记录到 commit message

### 每周（手动）

- 🔁 数据库 snapshot
- 🔁 永久资产打包（如果有新增）

### 每月（手动）

- 🧹 清理临时资产（爬虫图、老日志）
- 📊 检查外部服务用量（防账单突发）
- 🔐 两地 config.yaml 一致性校验

### 每季度（手动）

- 📝 整理 STATUS.md 历史日期（太老的可以压缩或归档）
- 🔍 Run 一次 10 题"万无一失"自检（见下一篇）
- 🎁 考虑新项目要不要建这套

## 💸 存储成本估算

典型个人项目：

| 资产 | 大小 | 费用 |
|------|------|------|
| 源码 GitHub | < 10MB | 免费 |
| GitHub LFS（大文件） | 1-10GB | $5/月（50GB 起） |
| iCloud Drive | 5-50GB | 免费 - ¥21/月 |
| 坚果云 | 1-10GB | 免费 |
| 阿里云 OSS | 100MB - 10GB | 极便宜（~¥1/月） |

**结论**：个人项目备份几乎免费。没理由不做。
