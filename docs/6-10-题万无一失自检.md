# 6. 10 题万无一失自检

> 搭完救命手册套件不代表真的"万无一失"。
>
> **自己跑一遍这 10 题**。任何一个答"不知道"或"应该吧"→ 还有工作要做。

---

## 🧪 10 题自检

### 题 1：本地 MacBook 磁盘坏了，代码能恢复吗？

**答案应该是**：能，GitHub 上有。

**验证**：
```bash
git log origin/main --oneline | head -3   # 远程有最近的 commit
git status --short                          # 本地没有未 push 的内容
```

**如果答不知道**：
- 检查是不是还没 push
- 检查 GitHub 网页能看到最新 commit

---

### 题 2：部署目标（Mac Mini / VPS / 云主机）硬盘坏了，服务能重建吗？

**答案应该是**：能，按 RUNBOOK.md § 4 从零重建。

**验证**：
- 打开你项目的 `RUNBOOK.md § 4`
- 确认每一步都有具体命令（不是"装依赖"这种模糊描述）
- `deploy/` 目录里有 LaunchAgent / systemd 文件
- 数据库 snapshot 在 `~/backups/`

**如果答不知道**：
- 补 RUNBOOK § 4 的具体步骤
- 做一次数据库 snapshot

---

### 题 3：`config.yaml` 全丢了（本地 + 部署目标），密钥能恢复吗？

**答案应该是**：能，`credentials.md` 里有完整副本。

**验证**：
```bash
cat ~/.claude/projects/*/memory/credentials.md | grep -c "项目名"
# 应该 >= 1
```
打开 credentials.md 看：有没有覆盖**所有**密钥字段。

**如果答不知道**：
- 去 credentials.md 补全今天项目的所有密钥
- 对照 `config.example.yaml` 逐字段检查

---

### 题 4：数据库数据丢了，能从 snapshot 恢复吗？

**答案应该是**：能，有 snapshot 在 `~/backups/`。

**验证**：
```bash
# 看最近 snapshot 时间
ls -lht ~/backups/ | head -5
```

**如果答不知道**：
- 现在做一次 snapshot
- 在日历上设"每周三做 snapshot"提醒

---

### 题 5：上一个 Claude 会话的心智状态，新 Claude 30 秒能进入吗？

**答案应该是**：能，贴开场口令或 `/<proj>-onboard`。

**验证**：
- 开一个新 Claude 窗口
- 贴开场口令
- Claude 读 ONBOARDING.md → 汇报状态
- 汇报内容对吗？

**如果答不知道**：
- 去更新 STATUS.md（当前阻塞、下次第一件事）
- 更新 ONBOARDING.md 的禁忌清单

---

### 题 6：今天之前的工作有 git 历史吗？

**答案应该是**：有，`git log --oneline` 能看到连续的 commit。

**验证**：
```bash
git log --oneline --since="1 month ago" | wc -l
# 应该 > 0
```

**如果答不知道**：
- 跑 `git log --oneline` 看看
- 如果几天没 commit：现在就 commit 一次
- 考虑设"每日会话结束必 offboard"规矩

---

### 题 7：手动命令我能记住吗？

**答案应该是**：不用记，都在 `scripts/deploy.sh` 里。

**验证**：
```bash
cat scripts/deploy.sh | head -20
# 看看自动化程度
```

**如果答不知道**：
- 把手动操作写进 `scripts/deploy.sh`
- 或者写进 `RUNBOOK.md § 2 日常部署`

---

### 题 8：外部依赖挂了我知道找谁吗？

**答案应该是**：RUNBOOK.md § 9 有外部依赖清单。

**验证**：打开 RUNBOOK.md § 9 看看：
- 每个依赖干啥
- 挂了会怎样
- 上哪找客服 / 官网

**如果答不知道**：
- 补 RUNBOOK.md § 9
- 特别是付费服务的到期日（防突然被停）

---

### 题 9：密钥泄露到 git 的风险？

**答案应该是**：不会。`.gitignore` 拦着，每次 commit 前 `git status --ignored` 验证。

**验证**：
```bash
# 1. .gitignore 有敏感文件
grep -E "config\.yaml|\.env|\.key" .gitignore

# 2. git 当前状态没敏感文件
git status --ignored --short | grep -E "config\.yaml|\.env|\.key"
# 这些应该在 !! 列（被忽略），不在 ?? 或 A 列

# 3. 历史里没敏感文件
git log --all --full-history --stat | grep -E "config\.yaml|\.env"
# 应该只有 config.example.yaml，没 config.yaml
```

**如果答不知道**：
- 补 `.gitignore`
- 检查历史（如果有泄露按 RUNBOOK § 10.9 处理）
- 轮换所有可能泄露的密钥

---

### 题 10：6 个月后的我能看懂当前状态吗？

**答案应该是**：能，`STATUS.md` + `RUNBOOK.md` 记录完整。

**验证**：
- 打开 STATUS.md，**假装你是 6 个月后的自己**
- 看得懂现在卡在哪吗？
- 知道下次怎么继续吗？

**如果答不知道**：
- STATUS.md 末尾的"下次第一件事"写得不够明确
- Phase 完成度表格描述太技术化

## 🎯 通关标准

- **10 题全答"能/确定"** → 🏆 真·万无一失
- **有 1-2 题不确定** → 🟢 基本到位，当天补齐
- **有 3-5 题不确定** → 🟡 中等风险，安排本周补
- **有 6+ 题不确定** → 🔴 高风险，现在就补

---

## 🔁 推荐频率

- **首次搭完救命套件**：立刻跑一遍
- **项目上线前**：再跑一遍
- **每季度**：跑一遍
- **重大变化后**（加了新服务 / 换了部署方式 / 删了老功能）：跑一遍
